Notes / Instructions:
	- To run the code use the facultyTable and studentTable txt files bundled along with the source code.
	- My outline for the student table is as follows:
		X - Indicates to the program that a new student object is about to be created
		# - Representes the student ID number
		ABC - String that represents the student's name
		ABC - String that represents the student's academic level
		ABC - String that represents the student's major
		#.# - Double that represents the student's GPA
		# - Int that represents the student's advisor
		Y - Indicates to the program that a student object is ready to be initialized and logged.

	- My outline for the faculty table is as follows:
		X - Indicates to the program that a new student object is about to be created
		# - Representes the faculty ID number
		ABC - String that represents the faculty's name
		ABC - String that represents the faculty's academic level
		ABC - String that represents the faculty's concentration
		# - Int(s) that represents the faculty's advisees
		Y - Indicates to the program that a student object is ready to be initialized and logged.
